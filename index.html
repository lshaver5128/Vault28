Connect Real-time Cloud Database

By default, Vault 28 runs in a local sandbox mode. Follow these steps to connect a free Google Firebase Database so users on other computers can submit cards and chat with you in real-time.

Step 1: Create a Firebase Project Go to firebase.google.com, sign in, and click Create a Project. Name it "Vault28".
Step 2: Create Firestore Database In the Firebase Console sidebar, click Firestore Database, then click Create Database. Choose Start in Test Mode (so read/write rules are open for testing) and select your database location.
Step 3: Register a Web App On your project overview page, click the web icon (</>) to register a web app. Name it "Vault28 Web".
Step 4: Copy API Config keys Firebase will show you a configuration script. Copy only the firebaseConfig object. It looks like this: 
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "vault28.firebaseapp.com",
  projectId: "vault28",
  storageBucket: "vault28.appspot.com",
  messagingSenderId: "123456...",
  appId: "1:123456..."
};
Step 5: Paste in app.js Open app.js in VS Code and paste your config into the FIREBASE_CONFIG variable at the very top of the file. Save and push the change to your GitHub repo!
